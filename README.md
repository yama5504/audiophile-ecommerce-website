# プレミアムオーディオECサイト（マルチページ・カンプ書き起こし実績）

実務を想定した大規模なデザインカンプ（複数画面・複雑な仕様）をもとに、フロントエンドの設計・実装・最適化をトータルで行った本格的なEコマースWebアプリケーションの開発実績です。
単なる見た目の書き起こしにとどまらず、Vanilla JSでの非同期コンポーネント設計、状態管理を伴うカートシステム、および多画面にわたるSass（Sass Modules）のモジュール設計など、実務の現場に即した高い設計思想のもと構築しています。

## 概要

### 要件・機能

4つの異なる画面（トップ、カテゴリ一覧、商品詳細、決済）にわたり、一貫したブランド価値を伝えるデザインをドット単位で忠実にトレース。さらに、以下の高度なフロントエンド機能を実装しました：

- **マルチページ（MPA）ナビゲーション & フルレスポンシブ設計:** スマホ・タブレット・PCの全環境でデザインが滑らかに追従。
- **動的カートシステム（ローカルストレージ連携可能）:**
  - 全画面共通のヘッダーからアクセス可能な「カートモーダル」機能。
  - 決済画面（`checkout.html`）での動的な注文サマリー（`renderCheckoutSummary`）の自動計算および、注文完了時の「サンクスモーダル（`ThanksModal`）」の制御。
- **UIコンポーネントの非同期インクルード:**
  ヘッダー、フッター、商品共通バナーなどの使い回すパーツを別HTML化し、JSで動的に挿入するコンポーネント指向の開発スタイル。

## 開発プロセス

### ディレクトリ構造

Viteのマルチページビルド仕様に則り、ルート階層に各画面のエントリーポイントとなるHTMLを配置。ソースコード側は「機能ごとのモジュール（JS/SCSS）」に細かく分離し、最終的に `main.js` と `style.scss` で統合する、クリーンでスケールしやすいディレクトリ構造を設計しています。

```text
.
├── .github/
│   └── workflows/           # GitHub Actions自動デプロイ設定
├── .gitignore               # Git管理除外設定（node_modules、distの除外）
├── README.md                # 本ドキュメント
├── index.html               # メインHTML（トップページ）
├── category.html            # カテゴリ一覧画面
├── product.html             # 商品詳細画面
├── checkout.html            # 決済・チェックアウト画面
├── package-lock.json        # 依存ライブラリのバージョンロック
├── package.json             # プロジェクト依存関係・ビルドスクリプト定義
├── preview.jpg              # README表示用のプレビュー画像
├── vite.config.js           # Vite設定ファイル（マルチページ用ビルド設定）
├── public/                  # 成果物にそのままコピーされる静的アセット
│   └── assets/              # 商品画像、ロゴ、各種アイコン等のSVG/画像群
└── src/                     # ソースコード開発ディレクトリ
    ├── main.js              # 全体統合・共通初期化ロジックを行うメインJS
    ├── js/
    │   ├── component-loader.js # 非同期コンポーネント読み込みエンジン
    │   └── cart-ui.js       # カートUI描画・計算・モーダル制御ロジック
    └── styles/
        ├── reset.css        # リセットスタイル
        ├── style.scss       # 各コンポーネントを統合するメインSCSS
        ├── _common.scss     # 変数や全画面共通の共通定義
        ├── _header.scss     # ヘッダー・ナビゲーション・カートモーダル
        ├── _footer.scss     # フッター
        ├── _top.scss        # トップページ専用スタイル
        ├── _category.scss   # カテゴリ一覧ページ専用スタイル
        ├── _product.scss    # 商品詳細ページ専用スタイル
        └── _checkout.scss   # 決済・入力フォーム・サンクスモーダル
```

### 使用技術・環境
- 言語・マークアップ: HTML5 / SCSS (Sass Modules対応) / Vanilla JS (ES6+ オブジェクト指向・モジュール分割)
- ビルドツール: Vite (マルチページ対応構成)
- 自動化環境: GitHub ActionsによるGitHub Pagesへの自動CI/CDパイプライン構築

### こだわった技術的アプローチ
1. async/await を駆使した自作コンポーネントローダーによる共通化設計
フレームワーク（React等）を使わない Vanilla JS 環境での複数画面（マルチページ）開発において、ヘッダーやフッター、パーツなどの「共通部分の二重管理」を防ぐため、Fetch API と async/await をベースにした自作のコンポーネントローダー（component-loader.js）を構築しました。
これにより、メンテナンス性を劇的に向上させるとともに、非同期処理の確実なシーケンス（読み込み完了後にUIイベントを初期化する流れ）を担保しています。

2. Vanilla JS による堅牢なEC状態管理とUI同期（カートシステム）
カート内の商品の増減、合計金額の計算、および別画面である決済ページ（checkout.html）へのデータの引き継ぎとサマリー描画（renderCheckoutSummary）など、ECサイトの核となるロジックをJSで自作しました。
関数ごとに「描画（UI）の更新」と「イベントの検知」の役割を疎結合に切り分け、実務のSPA（Single Page Application）開発に近い、クリーンで追いやすい状態管理（State Management）を意識してコードを組織化しています。

3. エラーハンドリングとDOMガードの徹底
マルチページ開発では、「トップページにはカートボタンがあるが、特定のページには存在しない」といった、画面ごとのDOMの有無が発生します。
本プロジェクトの main.js 内（setupCartToggle や setupMenuToggle 等）では、要素の存在チェック（if (!cartBtn || !cartModal) return）によるDOMガードを徹底。これにより、別画面に遷移した際に関数が存在しない要素を操作してブラウザ上にJavaScriptエラー（TypeError）を吐き、他の処理をストップさせてしまうリスクを100%排除しています。

4. @use 規則に準拠した Sass モジュール設計（関心の分離）
従来のグローバルな @import によるスタイルの衝突や意図しない上書きリスクを避けるため、モダンな Sass Modules（@use） を全面採用しました。
_header.scss や _checkout.scss のように画面やコンポーネント単位でスタイルファイルを完全に独立させ、メインの style.scss で統合管理。変数やミックスインもカプセル化された名前空間内で呼び出すことで、複数人開発や大規模開発にも耐えうる、安全で予測可能性の高いCSS設計を行っています。

### 今後の拡張計画
カートデータの永続化（LocalStorage連携）:
ブラウザを閉じたりリロードしたりしてもカートの中身が維持されるよう、ステートの変更を localStorage に自動同期させる機構を追加予定です。

入力フォームのバリデーション強化:
決済画面における住所、メールアドレス、クレジットカード番号（またはe-Money）の入力欄に対し、正規表現を用いた実務レベルのリアルタイムなエラーバリデーション（UX考慮型）の実装を計画しています。
